# Studio Khushi — Portfolio Website

A premium, editorial portfolio for a **Product & Surface Pattern Designer**.

The concept is **“The Swatch Archive.”** Because Khushi designs *repeats* and
*textiles*, every visual on the site is a **live, generative SVG seamless
pattern** rendered in the browser — no stock photography. Cards are styled like
tactile fabric swatches in a sample book, tagged with real domain vocabulary
(repeat type, colourways, file formats).

## Run it

It's a zero-build static site — just open it, or serve the folder:

```bash
# any static server works
python -m http.server 8000
# then visit http://localhost:8000
```

Serving over `http(s)` also enables the PWA service worker (offline caching).
Opening `index.html` directly by double-click works too — the service worker is
simply skipped on `file://`.

## What's inside

| File | Purpose |
| --- | --- |
| `index.html` | Full single-page site: hero, portfolio, pattern library, products, collections, about, services, testimonials, blog, Instagram, contact, footer |
| `project.html` | Case-study template (reads `?p=<Project Name>`): gallery, process, moodboard, palette, materials, mockups, result, PDF/share |
| `assets/js/patterns.js` | The pattern engine — deterministic SVG motif generators (floral, ikat, scallop, terrazzo, arches, …) |
| `assets/js/main.js` | Content data + rendering + all interactions |
| `assets/css/style.css` | Design system (tokens, type scale, components, dark mode, responsive) |
| `site.webmanifest`, `sw.js`, `robots.txt`, `sitemap.xml` | PWA + SEO plumbing |

## Design system

- **Palette:** White `#FFFFFF` · Warm Beige `#F6F2EC` · Soft Grey `#EFEFEF` · Charcoal `#222222` · Accent Gold `#C8A96A`
- **Type:** Fraunces (display serif) · Manrope (body) · Spline Sans Mono (swatch specs)
- **Motion:** page-load reveal, scroll reveals, hero parallax, hover zoom, custom cursor — all gated behind `prefers-reduced-motion`
- **Dark mode:** warm near-black theme, saved to `localStorage`

## Editing content

All copy and catalogue data live in plain arrays at the top of
[`assets/js/main.js`](assets/js/main.js) — `PROJECTS`, `PATTERNS`,
`COLLECTIONS`, `PRODUCTS`, `POSTS`, `QUOTES`, `SERVICES`. Add or edit an entry
and the grids regenerate automatically.

### Swapping in real photography

The swatch/product art is generated by `PatternEngine.makePattern(...)`. To use a
real image instead, replace the generated SVG inside `.swatch__art` /
`.product__art` with an `<img loading="lazy" src="…">` — the CSS hover-zoom and
layout already support `<img>`.

## Upgrading to the full stack (optional)

The brief lists Next.js + Sanity/Contentful + Cloudinary + an admin dashboard.
This build ships the complete **front-end experience** as fast static files so it
works and looks finished today. To make it CMS-driven later, port the section
markup into Next.js components and feed the same data arrays from your CMS — the
pattern engine and styles drop in unchanged.
